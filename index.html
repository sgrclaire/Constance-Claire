« interMatch » : documenter la rencontre par affinités culturelles

Ce web documentaire présente un prototype de site de rencontre qui ne part pas seulement de la géolocalisation ou de la photo, mais de ce que les individus écoutent, lisent et regardent. Il est réalisé dans le cadre du cours « Document numérique » (Master CTM).

1. Contexte et enjeux

Les plateformes de rencontre contemporaines reposent majoritairement sur un tri visuel et géographique. Or, une partie des utilisateurs expriment le besoin d’interactions plus qualitatives, fondées sur des affinités culturelles et cognitives. L’idée d’« interMatch » est de montrer qu’un rapprochement basé sur les goûts déclarés dans d’autres plateformes — musique, livres, films — peut constituer un meilleur point de départ pour la conversation.

Dans ce projet, nous faisons l’hypothèse que la médiation par les données culturelles issues d’API publiques ou documentées permet une rencontre plus « située » : on ne rencontre pas n’importe qui, on rencontre quelqu’un qui a écouté le même album, lu le même roman ou noté le même film sur Letterboxd.

Ce web documentaire décrit le dispositif, ses limites et la manière dont il est implémenté en HTML5 selon les consignes de l’enseignant.

2. Fonctionnement général du prototype

Le site est organisé en deux modes de mise en relation : le mode « qui se ressemble s’assemble » et le mode « les opposés s’attirent ». Le premier mettra en avant les profils partageant un grand nombre de références culturelles ; le second mettra en avant les profils complémentaires (par exemple quelqu’un qui lit beaucoup mais écoute peu de musique sera présenté à quelqu’un qui écoute beaucoup mais lit peu).

Techniquement, le mode sélectionné est reflété dans l’attribut de l’élément racine : <html data-match="similar"> ou <html data-match="opposites">. Cette information est ensuite stylée via la feuille de style afin de faire apparaître un thème clair ou sombre selon le mode choisi.

3. Intégration des API culturelles

Le dispositif s’appuie sur trois sources de données externes fréquemment utilisées par les publics culturels :

  • Spotify : récupération des playlists et des artistes les plus écoutés afin de déduire des « pôles d’intérêt musicaux ».
  • Babelio : consultation des bibliothèques de livres lus ou en cours ; ces données permettent de repérer les genres de prédilection.
  • Letterboxd : liste de films vus et notés, très utile pour rapprocher les cinéphiles.

Ces trois API sont utilisées ici de manière illustrative. Dans un contexte réel, il conviendrait de se conformer aux conditions d’utilisation et de ne stocker que les données nécessaires au matching.

4. Test de personnalité : « similarité » ou « opposition »

Pour répondre à la consigne de proposer une interaction avec le texte, nous intégrons un petit test de personnalité qui oriente le mode d’affichage. Le principe est le suivant : si l’utilisateur donne une majorité de réponses orientées vers la familiarité, le site bascule en « ressemblance » (thème clair) ; s’il exprime le besoin d’être challengé, le site bascule en « opposés » (thème sombre).

Cette partie démontre aussi la possibilité d’utiliser du JavaScript léger pour modifier l’état du document sans sortir du cadre HTML5.

« Quand je discute avec quelqu’un, je préfère partir de références communes. »

5. Données personnelles, droit d’auteur et ré-exploitation

Le recours à des données issues de plateformes culturelles pose la question du consentement et de la finalité. Dans le cadre de ce projet pédagogique, les données sont simulées. Dans un contexte de mise en production, il faudrait informer clairement l’utilisateur de la finalité (« proposition de rencontres »), de la durée de conservation et des droits d’accès.

De même, l’intégration d’images, de logos ou de contenus protégés doit se faire dans le respect du droit d’auteur. Pour ce web documentaire, nous recommandons l’utilisation de ressources libres de droit ou publiées sous licence Creative Commons.

6. Glossaire

Ce glossaire permet aux lecteurs de retrouver rapidement certains termes utilisés dans le document.

API
Interface de Programmation Applicative. Permet ici d’interroger un service comme Spotify ou Babelio pour récupérer des données culturelles.
Matching
Procédure de rapprochement entre deux profils, selon des critères définis (ici : goûts culturels et paramètres de personnalité).
Web documentaire
Forme éditoriale interactive publiée sur le web, mêlant texte, médias et navigation interne.
